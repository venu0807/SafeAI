## 🔍 CRITICAL IMPLEMENTATION DETAILS

### Spectral Subtraction (RESEARCH PAPER METHOD)
**Why it matters:** Per the research paper: "When eliminating stationary noise from the audio stream, we use a process called spectral subtraction." This is the paper's chosen noise reduction method.

**Implementation:**
```python
import librosa
import numpy as np

def spectral_subtraction_denoise(audio, fft_size=1024, noise_floor=0.01):
    """Apply spectral subtraction for noise reduction"""
    # STFT, estimate noise from initial frames, subtract
    noise_stft = librosa.stft(noise_segment, n_fft=fft_size)
    noise_mag = np.mean(np.abs(noise_stft), axis=1, keepdims=True)
    audio_stft = librosa.stft(audio, n_fft=fft_size)
    audio_mag = np.abs(audio_stft)
    magnitude = audio_mag - noise_mag
    magnitude = np.maximum(magnitude, noise_floor * np.max(audio_mag))
    denoised = librosa.istft(magnitude * np.exp(1j * audio_phase))
    return denoised
```

**Applied in preprocessing pipeline:**
```
1. Load audio
2. Pad/truncate
3. >>> SPECTRAL SUBTRACTION <<< (research paper method)
4. Pre-emphasis
5. MFCC extraction
6. Normalize
7. Output
```

### MFCC Feature Matching (Python ↔ Java)
**Parameters MUST match for Android integration:**
- **SAMPLE_RATE:** 16000 Hz
- **DURATION:** 1.0 second
- **N_SAMPLES:** 16000
- **N_MFCC:** 40 coefficients
- **N_FFT:** 2048
- **HOP_LENGTH:** 512
- **N_MELS:** 128
- **MAX_TIME_STEPS:** 100
- **Output shape:** (100, 40)

**Python Extraction:**
```python
mfcc = librosa.feature.mfcc(
    y=audio,              # Audio signal
    sr=16000,             # Sample rate
    n_mfcc=40,            # Coefficients
    n_fft=2048,           # FFT size
    hop_length=512,       # Hop
    n_mels=128            # Mel bands
)
mfcc = mfcc.T             # Transpose to (time, features)
mfcc = (mfcc - mean) / std  # Normalize per feature
```

### Normalization (Per-Feature, Column-wise)
**INCORRECT (per-sample):**
```python
mfcc = (mfcc - np.mean(mfcc)) / np.std(mfcc)  # ✗ WRONG
```

**CORRECT (per-feature):**
```python
mfcc = (mfcc - np.mean(mfcc, axis=0, keepdims=True)) / (np.std(mfcc, axis=0, keepdims=True) + 1e-8)  # ✓ RIGHT
```

### Model Input Shape
- **Input:** (batch_size, 100, 40, 1)
  - 100 = MAX_TIME_STEPS (time dimension)
  - 40 = N_MFCC (feature dimension)
  - 1 = channel (grayscale-like)
- **Output:** (batch_size, 2)
  - [prob_normal, prob_distress]

## 🔧 TROUBLESHOOTING

### Issue: Low Accuracy (< 85%)

**Causes & Solutions:**
1. **Spectral subtraction disabled** → Check USE_SPECTRAL_SUBTRACTION in config.py
2. **Wrong MFCC parameters** → Verify match Android MFCCExtractor.java
3. **Normalization per-sample instead of per-feature** → Fix to column-wise
4. **Imbalanced dataset** → Ensure class_weight=True
5. **Dataset quality** → Check audio files are valid, not corrupted
6. **Too few samples** → Need minimum 2000 samples (1000 per class)

### Issue: Memory Error During Training

**Solutions:**
1. Reduce BATCH_SIZE in config.py (try 16 instead of 32)
2. Reduce EPOCHS (try 30 instead of 50)
3. Use GPU (TensorFlow will auto-detect)

### Issue: Files Not Found

**Causes:**
1. Audio files not in `ml/processed_data/`
2. Metadata.csv not generated → Run `1_organize_datasets.py` first
3. Preprocessed features not saved → Run `2_preprocess_audio.py`
